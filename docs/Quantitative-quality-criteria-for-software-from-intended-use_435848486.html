1. Purpose & Applicability

The manufacturer has comprehensibly derived quantitative quality criteria from the intended use of the software.
These criteria are linked in a traceability matrix to demonstrate conformity with:
- ISO 13485, §7.3.3 (design input requirements must be clear, testable, and based on intended use)
- IEC 62304, §5.2 (software system requirements)
- FDA Software Validation Guidance, 5.2.2 (quantitative requirements linked to verification)
- Xavier University Paper: Building Explainability and Trust for AI in Healthcare (trust is supported by transparency of performance metrics).

2. Intended Use (Prototype)

The software is intended to assist in the early detection of cerebral palsy (CP) risk in infants by analyzing motor patterns from video.
It is not a stand-alone diagnostic tool, but a decision support system for clinicians.

3. Chosen Operating Point

  • The AI model was evaluated at several thresholds (see external validation study, eTable 5).
  • Cutoff 0.35 has been selected for use in the prototype software, as it provides a balanced trade-off between:
    • Sensitivity (≥70% detection of CP risk infants, minimizing missed cases)
    • Specificity (>90%, minimizing false alarms)
    • High Negative Predictive Value (≈95%, ensuring that infants predicted “no CP” are reliably safe)
    • Acceptable Positive Predictive Value (≈68%, supporting trust in positive predictions).

5. Compliance Statement (Prediction model only)

The quantitative requirements defined above are:
- Derived directly from the intended use (supporting early CP risk detection).
- Traceable to measurable metrics (sensitivity, specificity, PPV, NPV, accuracy).
- Verified by external validation evidence (eTable 5).
- Aligned with applicable standards (ISO 13485, IEC 62304, FDA guidance, Xavier University paper).

This fulfills the requirement that the manufacturer must comprehensibly derive quantitative criteria from the intended use and represent them in a traceable manner.
