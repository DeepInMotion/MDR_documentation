Deep In Motion : 14.5 Data storage "Bucket"

Created by Felix Tempel, last updated on 13.06.2025

Documentation:

Context in (Groos et. al)

  • Although most infants had a single video used, if multiple recordings existed for the same infant, a rule was applied: use the recording between 12 and 13 weeks corrected age if available; if multiple, select the standardized-timepoint video. This ensures each infant contributes only one sample, avoiding splitting the same infant’s data across sets.

  • In cases where an infant had recordings at two timepoints (7 & 8 weeks), both were included only if they met inclusion criteria, but in general, per-infant grouping was enforced.

  • No infant appears in both the development and test sets.

  • In the implementation, the mapping is stored in a file for traceability.

  • The coordinate files are stored as .npy files.

  • The random seed is recorded.

Justification

  • Prevents data leakage: if the same subject’s data appeared in train and test, the model could overfit to idiosyncratic patterns.

  • Maintains independence between sets for valid performance evaluation.

Regulatory References

  • While no specific standard clause cites this directly, ISO 13485 design and validation planning would require describing how data independence is maintained to ensure valid performance evaluation.
