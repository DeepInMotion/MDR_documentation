Deep In Motion : 14.3 Data splitting procedure

Created by Felix Tempel, last updated on 11.06.2025

Documentation

Reported Split

  • In (Groos et. al.),Â data for 557 infants were split as follows:

    • Development set (for training and internal validation): 75.0% of infants.

    • External validation (test) set: 25.0% of infants.

  • Within the development set, 7-fold cross-validation was used for internal validation, with each fold comprising a balanced subset (e.g., each fold ~9 CP cases and ~50 non-CP cases).

Justification of Ratios

  • Statistical power & representativeness: 75/25 split ensures sufficient data in development to train deep models and perform internal validation, while reserving a sizable independent test set for unbiased external validation.

  • Maintaining prevalence: Splits stratified such that CP prevalence in each set mirrors overall prevalence, ensuring comparability of performance metrics.

  • Resource constraints: Deep learning benefits from more data in training; 75% for development balances model capacity needs with the necessity of an independent test set.

  • Randomization procedure: random assignment within strata (see 14.4).
