Deep In Motion : 7.2 UI behavior when output does not meet quality criteria

Created by Pål Haugen on 18.08.2025

The prototype also accounts for situations where outputs cannot meet the defined quality criteria. If predictions fail or are incomplete, the interface does not present empty visuals or uncertain values. Instead, it falls back to clear placeholder text such as “No data available”. This approach avoids misleading the user with results that appear valid but are based on insufficient or low-quality data. By preventing the display of unreliable outputs, the system ensures that users can trust the results they see on screen and are alerted when outputs are not usable.
