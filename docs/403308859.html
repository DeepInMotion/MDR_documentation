Regulatory references

12.1

The manufacturer has set a procedure that describes the pre-processing of the data.


ISO 13485:2016, 4.1.6, 7.3.6, XAVIER University "Building Explainability and Trust for AI in Healthcare", FDA 21 CFR part 820.70(i)

12.2

This procedure describes the individual processing steps such as conversion, transformation, aggregation, normalization, format conversion, calculation of feature, conversion of numerical data into categories.

A graphic representation creates a rapid overview. The conversion of numerical to categorical values requires a justification.


12.3

The procedure describes how the correctness of the interim steps and the final results are assessed15 through risk-based evaluations.

This is consistent with the requirements of ISO 13485:2016 chapter 4.1.6. The risk management file must contain these analyzes.

ISO 13485, 4.1.6., 7.3.2., 7.5.6.

12.4

This procedure specifies how values with various measurement scales or units are detected and processed.

This depends on the ML-method used (e.g. tabular data / image data) and cannot be demanded as a general best practice.


12.5

This procedure specifies how values are detected and processed that have been collected with various measurement methods.

This depends on the ML- method used (e.g. tabular data / image data) and cannot be demanded as a general best practice.


12.6

This procedure specifies how values or metadata with the same names (such as in column headers) are detected and processed.

This, however, depends on the ML-method used (e.g. tabular data / image data) and cannot be demanded as a general best practice.


12.7

This procedure specifies how missing values within data sets are detected and processed. The manufacturer gives a rationale for the decision16.

Make sure that the rationale differentiates between “missing at random” and “missing not at random”17.


12.8

This procedure specifies how outliers are detected and processed18. The manufacturer gives a rationale for the decision19.

Show example of a date / feature. This, however, depends on the ML-method used (e.g. tabular data / image data) and cannot be demanded as a general best practice.


12.9

This procedure specifies how unusable data sets are detected and handled20. The determination was justified by the manufacturer.

Request example of a date / feature.


12.10

The manufacturer has identified, assessed, and managed the risks arising from data processing.

Risks can be caused, for example, by software errors, rounding errors, re-sampling and compression of data, and the sorting out of invalid data.
