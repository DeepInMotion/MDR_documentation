Deep In Motion : 11.2: Data Labeling Procedure

Created by Daniel Groos, last updated on 19.08.2025

Requirements

  • The manufacturer has defined a procedure for labeling in "Supervised Learning", if labels were not yet present in the data.

Summary

A controlled procedure was defined to generate clinically valid labels for supervised learning when these were not originally present in the raw data.

Labeling Procedure

  1. Data–Record Linkage

    • Raw video recordings were linked to the infant’s clinical record via pseudonymized identifiers.

    • Only after secure linkage were outcome data accessed for labeling.

  2. Clinical Outcome Determination

    • Labels were derived from follow-up diagnoses performed by qualified clinicians using standardized CP diagnostic criteria.

    • Infants without CP diagnosis at follow-up were labeled as non-CP.

  3. Independent Verification

    • Ambiguous cases were reviewed by at least two independent clinicians.

    • Disagreements were resolved by consensus to ensure reliability.

  4. Documentation and Audit Trail

    • Each labeling decision was logged with reference to source documents (e.g., clinical report ID, assessment date).

    • An auditable trace from raw data → clinical record → assigned label was maintained.

Justification

  • Ensures labels are clinically accurate, standardized, and reproducible.

  • Prevents mislabeling and supports traceability of every data point used in supervised learning.

  • Aligns with MDR expectations for transparent linkage between intended use, raw data, and model training.
