Deep In Motion : 7.1 UI behavior when preconditions are not met

Created by Pål Haugen, last updated on 18.08.2025

When the preconditions for safe operation are not fulfilled (e.g., invalid or missing input files), the prototype UI clearly communicates this to the user. Instead of showing partial or misleading results, the system displays explicit notifications such as “No data available” or error messages explaining the issue. For example, if a video is missing, corrupted, or incompatible, the dashboard prevents the risk calculation and informs the user of the problem. This ensures that the system fails safely and does not present outputs that could be misunderstood as valid clinical information.
