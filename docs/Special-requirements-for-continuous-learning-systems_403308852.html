Regulatory references

9.1

The manufacturer has defined when and how often the model is updated.

For this purpose, the manufacturer should define the "triggers" of these updates. He must also describe whether these updates take place per individual medical device or for all medical devices of the type.


9.2

The manufacturer has specified how the additional data will be quality assured and how incorrect, missing or implausible data will be dealt with.



9.3

The manufacturer has defined in which range the output data may change.

This requires a description of how the algorithms change.


9.4

The manufacturer has described how it continues to ensure the correctness and accuracy of the output data and what happens if this can no longer be guaranteed.

Self-checks, the possibility of roll-backs, limiting outputs are possible approaches.


9.5

The manufacturer has described whether and how users are informed about changes to the algorithm and whether they can decide about them.



9.6

The manufacturer has identified and managed the specific risks arising from continuous learning.

To this end, the manufacturer must justify that it achieves a better risk-benefit ratio through continuous learning.
