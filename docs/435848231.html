Deep In Motion : 10.10: Examination of Potential Label Leakage

Created by Daniel Groos, last updated on 19.08.2025

Requirements

  • The manufacturer has examined and excluded the possibility of a “label leakage”.

Summary

Systematical evaluation of the data and analysis pipeline have been performed to ensure that no outcome-related information (labels) was inadvertently included in the input data used for model training or external validation.

Procedure

  • Data Review: Verified that input data (video recordings of infant movements) contained only raw observational content, free from clinical annotations or metadata that could reveal diagnosis.

  • Metadata Control: All diagnostic information, clinical notes, and outcome data were stored separately from input data.

  • Cross-Validation Safeguards: Strict separation of development and external validation cohorts ensured that no diagnostic information was leaked between data subsets.

Justification

  • Prevents artificial inflation of performance metrics.

  • Ensures that algorithmic predictions are based solely on movement features observable in real-world application.

  • Aligns with best practices for trustworthy AI in medical devices and with MDR requirements for reproducibility.
