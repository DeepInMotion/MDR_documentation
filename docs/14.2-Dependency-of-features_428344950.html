Deep In Motion : 14.2 Dependency of features

Created by Felix Tempel, last updated on 11.06.2025

Description of Feature Dependencies

  • In video-based skeletal data, the 19 body points form a known anatomical graph: joints are connected according to the human body’s kinematic chain (e.g., the elbow connects the shoulder and wrist). The deep learning architecture (i.e, graph convolutional network) leverages this topology to model dependencies among joint movements.

  • Temporal dependencies: joints’ positions/velocities across frames/windows are temporally correlated; the model processes sequences (i.e, using temporal convolution with attention mechanisms) to capture how current movement depends on past frames.

  • Biomechanical relationships: segment lengths tie pairs of joint coordinates; e.g., distance between shoulder and elbow is constrained by anatomy. These constraints inform that certain features are interdependent and reduce spurious correlations.

  • The model is designed to exploit dependencies within the features to learn complex movement patterns.
    • The model processes contiguous 5-second windows, capturing correlations over time between joint movements and the features derived therefrom.
    • The features are fused in one main stream block after their individual input stream block.

Visualization / DAG

  • The body key points are embedded in an adjacency matrix, which has different aggregation capabilities, which are defined in (Yan et. al).

  • Nodes: each joint coordinate or biomechanical property.

  • Edges: anatomical connections (e.g., shoulder ↔ elbow, elbow ↔ wrist).

  • Temporal edges: connect same joint across successive time steps/windows.
