Deep In Motion : 17.1 Version control

Created by Felix Tempel, last updated on 19.06.2025

All code used for data preprocessing, training, architecture search (NAS), and explainability was maintained under Git-based version control.

The public codebases are available at:Â https://github.com/DeepInMotion/

Control procedures:

  • Commit logs are used to track each development stage.

  • Model checkpoints were versioned alongside corresponding config files.

  • Training and evaluation scripts, as well as infrastructure Docker files, are included in the repository.

  • The coordinate files are saved as .npy files on a private server with restricted access.
